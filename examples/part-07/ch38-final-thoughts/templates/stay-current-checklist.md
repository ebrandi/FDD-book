# Stay-Current Checklist

Use this checklist on a monthly cadence to keep yourself in
step with FreeBSD development. It is deliberately short; a
checklist that takes an hour is more likely to get done than
one that takes a full day.

**Month:** _________

**Date completed:** _________

---

## Source Currency

- [ ] I have pulled the latest FreeBSD source tree.
- [ ] I have rebuilt the kernel on my main development
  machine against the latest sources.
- [ ] I have built my personal drivers against the latest
  source and investigated any new warnings or errors.
- [ ] I have run my regression tests against the latest
  source and investigated any failures.

## Release-Notes and UPDATING

- [ ] I have read the UPDATING file for any new notices since
  my last check.
- [ ] If a recent release branched, I have skimmed the
  release notes.
- [ ] I have noted any items that affect my ongoing work.

## Commit and Subsystem Awareness

- [ ] I have read the commit log for the subsystems I care
  about since the last time I looked.
- [ ] I have identified at least one commit worth reading in
  detail and read it.
- [ ] I have traced any API changes that intersect with my
  drivers.

## Community

- [ ] I have read at least one thread on `freebsd-hackers@`
  or `freebsd-drivers@`.
- [ ] I have watched at least one conference talk or read at
  least one blog post about FreeBSD.
- [ ] I have replied on a mailing list or a review at least
  once this month.

## Reflection

- [ ] I have thought about whether any of the changes I saw
  affect my ongoing work.
- [ ] I have noted any TODO items for next month's review.
- [ ] I have saved this filled-in checklist in my
  `freebsd-learning` repository.
